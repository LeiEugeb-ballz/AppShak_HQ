Overview

AppShack produces 2–10 minute, human-like audio recordings of multi-agent debates or roundtable discussions. A “chairperson” user selects a topic and 2–6 personas, tweaks tone/expertise, then triggers generation. The system orchestrates an LLM for dialogue, TTS for voice, stitches audio, and stores it in a library for streaming or download.

User Journey

  • Login/Onboard: OAuth/email/SAML; store preferences.
  • Create Session: Enter topic; choose personas; adjust traits.
  • Generate Debate: LLM → transcript → TTS → audio.
  • Review & Publish: Preview snippet; approve; tag; save to library.
  • Stream/Download: Browse library; filter by topic/persona; play HLS or download mp3.

System Architecture

Browser ←HTTPS→ API Gateway
    └─ Backend Services (Node.js/Python)
       ├─ Auth Service
       ├─ Session Manager
       ├─ LLM Orchestrator (OpenAI GPT-3.5-turbo)
       ├─ TTS Orchestrator (e.g., AWS Polly)
       ├─ Media Processor (FFmpeg)
       ├─ Library + Metadata DB (DynamoDB)
       └─ Streaming (CloudFront/S3)

Data Flow

  1. User → Backend: POST /api/session with { topic, personas }.
  2. Backend → OpenAI: system + user prompts → transcript with speaker labels.
  3. Transcript → TTS API per speaker chunk → audio clips.
  4. Media Processor concatenates and encodes to mp3.
  5. Audio + metadata stored in S3; metadata in NoSQL DB.
  6. Frontend receives { sessionId, audioUrl, transcript }.

Cost Estimation

Total cost per audio: $$\text{Cost}_{audio} = \text{Cost}_{LLM} + \text{Cost}_{TTS} + \text{Cost}_{storage}$$

  • $\text{Cost}_{LLM} = \$0.004$ (2,000 tokens @ \$0.002/1k)
  • $\text{Cost}_{TTS} = \$0.012$ (3,000 chars @ \$4/1M)
  • $\text{Cost}_{storage} = \$0.0001$ (5 MB @ \$0.023/GB-mo)

Total: $\$0.0161$ < \$1 per session.

Implementation Roadmap
  1. Infra Setup: API gateway, containers, S3, NoSQL, env vars.
  2. Backend Skeleton: Models, Auth, Session & Library endpoints.
  3. LLM Integration: Prompt templates, OpenAI SDK, streaming chat.
  4. TTS Integration: Provider selection, chunking, rate-limit handling.
  5. Media Processing: FFmpeg for concat & encode; preview generation.
  6. Frontend: Persona selector, progress UI, library & player.
  7. Testing & QA: Unit and E2E tests.
  8. Monitoring: Logs, cost alerts, retry logic.
Security & Compliance
  • Store API keys in env or Secrets Manager.
  • HTTPS/TLS for all endpoints.
  • Least-privilege IAM for S3/DB.
  • Encryption at rest (S3) & in transit (TLS).
  • GDPR/CCPA: data export/purge on request.
Non-Functional Requirements
  • Latency: <30 s to full audio after trigger.
  • Scalability: auto-scaling backend.
  • Reliability: retry on transient errors.
  • Cost Control: per-session spend tracking & caps.
